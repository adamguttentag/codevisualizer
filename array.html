Help

Basics

  1. Read the prompts in the message box on the left side of the window.
  2. Type your code in the console box near the bottom of the window and press enter.
  3. Watch the animation play out in the middle of the window.

Mouseover Help

If you're not sure what something is or does, move your cursor over it. An icon will appear below the console with helpful information. Objects in the visualization will tell you what their properties are. Interface elements like dock buttons, progress bars and the message box will tell you what they do. Be inquisitive!

Console History

The console saves your history, just like the browser's JavaScript console or a terminal window. When the console has focus, pressing the up key on your keyboard fills the console with the last text you entered. You can scroll back and forth through your history with the up and down keys.

Decluttering Your Workspace

If you have a lot of variables on screen, some may be covered by the message box. To temporarily slide the message box out of your way, click or tap it. You can always bring it back into view by clicking or tapping on the tab that remains visible. It will slide back out when it has new information for you.

There isn't really a way to "delete" a variable in JavaScript (built-in garbage collection releases variables out of scope for you), but if you want to remove a variable from Code Visualizer, you can undefine it: `a = undefined` would remove the variable `a`.

Errors

Error messages may appear in the console window if you try to do something silly like push() a variable that doesn't exist.

Yellow alerts advise you that your code is valid but may not produce the results you expected.

Red error messages suggest you may have entered invalid code.

Pocket Guide

The Pocket Guide contains information about concepts you have learned. It's your own personalized reference guide and trophy case.

The more you practice a concept, the more detail your Pocket Guide will display about that concept. This helps you commit the concept to memory through repetition, but also gives you a handy way to recall the details later on.

Click the Pocket Guide icon in the bottom right corner of the window to open it. Page numbers at the top of the guide indicate where you are in the guide and how many pages the guide contains. If there are more than 2 pages, click or tap the or icons to turn the pages.

In the future (as in, once I implement the code to arrange the guide in booklet format for printing), you will be able to print your personalized Pocket Guide so you have a hard copy you can take anywhere. Once the Pocket Guide is open, click the printer icon to print it. If you're using Chrome, you can select "Save As PDF" in the print dialog to create a file that can be viewed on your computer, phone or tablet.

Settings

You can change the background image by entering the URL of a different image. Search Google, Dribbble or Behance for "landscape" and find something inspiring to serve as a backdrop for your learning. Right-click the image, copy the image URL, paste it into the background image input box and press tab. Here are some that caught my eye:

  • MRusta's "Mountain" (animated)
  • Louis Coyle's "Lakeside Sunset"
  • Dave Chenell's "Silent Escape" (animated)
  • Ilari Tuupanen's "KOLI"
  • Daniel Řeřicha's "Spring South Moravia"
Out of respect for the artists, these links go to their portfolio page, not directly to the image.

As this app is a proof of concept demo, you can "cheat" and fill the Pocket Guide's pages using the debug menu in the settings window. Use the select elements to change the content level for each of the three concepts covered in the demo, then click on the Pocket Guide icon to view the pages.
